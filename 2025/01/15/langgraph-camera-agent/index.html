基于 LangGraph 构建智能相机调优 Agent

发表于 2025-01-15 更新于 2026-06-04 分类于 技术

在高通的项目中，我用 LangGraph 构建了一套多智能体相机调优系统，将 ISP 参数迭代周期从数天压缩到数小时，效率提升约 60%。本文介绍系统架构、关键技术与实际效果。

背景

相机 ISP 参数调优是一个高度依赖经验的复杂过程。传统方式下，工程师需要手动调整数百个参数，每次迭代耗时数天，且效果很大程度上依赖个人经验积累。

核心痛点：

  • 参数空间庞大，人工搜索效率低
  • 历史调参经验难以复用
  • 评估周期长，反馈慢

系统架构

整个系统采用多智能体协作模式，核心流程如下：

1. 规划智能体（Planner）

接收用户意图（如”提升暗光噪点表现”），结合 RAG 检索历史调参案例，生成可执行的调参计划。

2. 执行智能体（Executor）

根据计划调用相机测试工具链，生成对应配置并触发图像采集。

3. 评估智能体（Evaluator）

对采集图像进行质量评估（PSNR、SSIM、NR 质量等指标），判断是否满足要求。如果不满足，将反馈送回 Planner 进行策略调整。

关键技术点

RAG 增强规划

我们构建了一个 ISP 参数知识图谱，使用 LangChain 的 VectorStore 存储历史调参方案。Planner 在生成计划前，先通过语义检索召回最相关的历史案例，显著提升规划质量。

安全护栏

对于 ISP 参数这类敏感操作，系统设置了安全限制：

  • 单次参数调整幅度不超过阈值的 10%
  • 异常值检测与自动回退
  • 关键步骤人工确认

效果

上线后在多个相机场景下进行了验证：

场景 人工迭代周期 Agent 迭代周期 效率提升
暗光噪点 3 天 1 天 67%
色彩还原 5 天 2 天 60%
运动模糊 4 天 1.5 天 62%

总结

LangGraph 为构建复杂的多步骤 Agent 系统提供了优雅的状态管理方案。在相机调优这个垂直领域中，我们验证了 Agent + RAG + 工具链 的可行性和高效性。

几点经验：

  1. 状态设计先行：StateGraph 的状态结构决定了整个系统的信息流，要在设计阶段想清楚
  2. RAG 质量是瓶颈：历史案例的质量和检索精度直接影响 Planner 的规划效果
  3. 安全护栏不可少：自动化调参必须有回退机制，避免参数越调越差

未来计划将这套框架扩展到更多 ISP 调优场景，并探索与强化学习的结合。
